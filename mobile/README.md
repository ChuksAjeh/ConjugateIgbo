# Conjugate Igbo Mobile

Expo React Native app for learning Igbo verb conjugations.

## Project Structure

```text
app/                    Expo Router screens and navigation groups
app/(tabs)/             Main tab screens: practice, verbs, favourites, pro, settings
components/             Shared React components and app-level providers
constants/              Shared design tokens and static constants
data/                   Local verb dataset
hooks/                  Reusable stateful logic for settings, purchases, notifications, etc.
lib/                    Services, platform integrations, and pure domain logic
lib/dialects/           Dialect-specific conjugation rules
models/                 Shared TypeScript domain models
styles/                 Screen-specific style factories
assets/                 Icons, splash images, and app artwork
__tests__/              Jest tests and snapshots
ios/ android/           Native projects generated by Expo prebuild/run commands
```

## Key App Flows

- `app/_layout.tsx` initializes Sentry, fonts, onboarding state, app providers, and verb preloading.
- `components/PurchasesProvider.tsx` owns RevenueCat customer state, offerings, purchase, and restore logic.
- `lib/purchaseProducts.ts` defines the lifetime Pro product and validates that the selected offering is not a subscription.
- `lib/conjugateVerbs.ts` and `lib/dialects/*` contain the conjugation engine.
- `lib/responsive.ts` centralizes tablet/large-screen sizing decisions.

## Logging And Sentry

- Sentry is initialized once in `lib/sentry.ts`.
- App code should use `logger` from `lib/logger.ts` instead of `console.*` or direct `Sentry.logger.*`.
- Use structured context on every log: `feature`, `component`, `tags`, and `extra`.
- Use `logger.info` for successful lifecycle events, `logger.warn` for recoverable abnormal states, and `logger.error` for failures that should be visible in Sentry.
- Native crashes require the Sentry native SDK plus matching uploaded dSYMs/source maps. For EAS builds, keep `SENTRY_AUTH_TOKEN` configured in the build profile environment.

Example:

```ts
logger.error(error, 'Failed to restore purchases', {
  feature: 'purchases',
  component: 'PurchasesProvider',
  extra: { context: 'Restore failed' },
});
```

## Purchases

Conjugate Igbo Pro is a lifetime one-time payment.

- App Store Connect product type: non-consumable.
- RevenueCat product id: `conjugate_igbo_pro`.
- RevenueCat package: lifetime (`$rc_lifetime`) or a custom package that maps to the same non-subscription product.
- RevenueCat entitlement: `ConjugateIgbo Pro`.
- Required EAS/App Store build env var: `EXPO_PUBLIC_REVENUECAT_API_IOS_KEY`.

Do not configure this product as an auto-renewable subscription, non-renewing subscription, or consumable. The app intentionally refuses to select subscription-like packages for Pro.

## Release Checklist

1. Confirm `SENTRY_AUTH_TOKEN` exists in the EAS production profile environment.
2. Confirm `EXPO_PUBLIC_REVENUECAT_API_IOS_KEY` exists in the EAS production profile environment.
3. Confirm RevenueCat offering includes the lifetime non-consumable product `conjugate_igbo_pro`.
4. Run `npm run lint`, `npx tsc --noEmit`, and `npm test -- --runInBand`.
5. Submit a fresh build number after native/Sentry/purchase changes so Apple Review is not testing an old binary.
